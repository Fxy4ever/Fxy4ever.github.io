5.版本回退

在Git中用，HEAD 表示当前版本，HEAD^ 计算上一个版本，HEAD^^ 就是上上个版本， HEAD～100就是往上100个版本。

reset是直接去除当前这个节点，回退到上一个节点。

此时git log，会发现 没有回退之前的版本信息了。

如果后悔了怎么重新回到之前的版本？—当然是顺着命令行往上找到那个版本的id

6.工作区和暂存区
工作区（Working Directory）

就是你在电脑里能看到的目录

版本库(Repository)

ps:如果在git add之后修改文件，再git commit 则不会有什么变化。 因为此时修改的是工作区的文件，而git commit提交的文件是暂存区的。



所以说正确的方式应该是
:第一次修改 -> git add -> 第二次修改 -> git add -> git commit
